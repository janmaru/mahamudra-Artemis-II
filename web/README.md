# Artemis II Web Dashboard

A responsive web version of the Artemis II mission dashboard, deployable on GitHub Pages.

## Features

### Current (Phase 1)
- ✅ Real-time MET (Mission Elapsed Time) display
- ✅ Spacecraft telemetry (Earth/Moon distance, speed, position)
- ✅ Deep Space Network communications status
- ✅ Space weather from NOAA SWPC
- ✅ Solar flares & CME alerts from NASA DONKI
- ✅ Photo carousel (10 photos from NASA IOTD, rotation every 6 seconds)
- ✅ Canvas trajectory map (Earth/Moon/Orion with logarithmic scaling)
- ✅ Terminal-style UI (green-on-black theme)
- ✅ Responsive layout (desktop & mobile)
- ✅ GitHub Actions auto-refresh (every 10 minutes)

## Architecture

### How it Works

1. **GitHub Action** (`.github/workflows/update-web-data.yml`) runs every 10 minutes
2. Action runs `scripts/update_web_data.py` which fetches fresh data from NASA APIs
3. Telemetry data saved as JSON files to `web/data/` (spacecraft, DSN, weather, alerts)
4. Photos fetched directly from NASA IOTD RSS feed (up to 10) and saved to `web/data/photos.json`
5. Changes are committed and pushed to the repository
6. GitHub Pages serves the static HTML + cached JSON
7. Browser loads `index.html`, reads the cached JSON, and rotates photos every 6 seconds

### Files

- **index.html** - Main dashboard layout
- **dashboard.js** - UI controller, canvas trajectory map, photo carousel (6s rotation)
- **api-client.js** - Reads cached JSON from `data/` directory
- **styles.css** - Terminal-inspired styling
- **server.py** - Local development server with CORS proxy (port 8000)
- **data/** - JSON cache files (auto-generated by GitHub Actions)
- **data/photos.json** - Array of up to 10 NASA IOTD photos (title, image_url, published)
- **.github/workflows/update-web-data.yml** - GitHub Action configuration
- **scripts/update_web_data.py** - Fetch script (runs in GitHub Actions)

## Data Flow

```
NASA APIs (Horizons, DSN, SWPC, DONKI)
    ↓
GitHub Action (every 10 min)
    ↓
scripts/update_web_data.py
    ↓
web/data/*.json (telemetry + photos.json with 10 images)
    ↓
GitHub Pages
    ↓
Browser loads data from web/data/
    ↓
Dashboard: canvas trajectory map + photo carousel (6s rotation)
```

## Local Development

```bash
# Run the local test fetch script
python scripts/update_web_data.py

# Serve the dashboard
python -m http.server 8000 -d web
```

Then visit `http://localhost:8000`

## Deployment to GitHub Pages

1. Push this repository to GitHub
2. Enable GitHub Pages in Settings → Pages (source: `main` branch, `/root`)
3. GitHub Actions will automatically run on push and every 10 minutes
4. Data files will be created/updated in `web/data/`
5. Visit `https://yourusername.github.io/mahamudra-Artemis-II/`

## GitHub Action Setup

The workflow file `.github/workflows/update-web-data.yml`:
- Triggers every 10 minutes (`*/10 * * * *`)
- Installs Python dependencies
- Runs `scripts/update_web_data.py` to fetch fresh NASA API data
- Commits and pushes updated `web/data/*.json` files

No additional setup needed - just push to GitHub!

## Data Sources

All data comes from public NASA APIs (no authentication required):

| Source | Data | Updated |
|--------|------|---------|
| **Horizons** | Spacecraft position/velocity | Every 10 min |
| **DSN** | Deep Space Network comms | Every 10 min |
| **SWPC** | Space weather | Every 10 min |
| **DONKI** | Solar flares, CMEs | Every 10 min |
| **NASA IOTD RSS** | Mission photos (10 images) | Every 10 min |

## Browser Requirements

- Modern browser with Fetch API support (all modern browsers)
- Works offline with cached data

## Troubleshooting

**Dashboard shows "Loading..." forever?**
- Check browser console (F12) for errors
- Ensure `web/data/` directory has JSON files
- Check GitHub Actions tab for workflow failures

**Data is stale?**
- Trigger GitHub Action manually: Actions tab → "Update Artemis II Web Dashboard Data" → "Run workflow"
- Or wait for next scheduled run (10 min)
